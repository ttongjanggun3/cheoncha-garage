# 🏎️ 천차 Garage (CheonCha Garage) 🏎️ (제출물 4형태 최종 README)

## 무엇을 하는 프로그램인가요?

천차 Garage는 자동차 제작 동아리 **천차많별**의 출석 관리를 자동화하기 위한 Python 프로그램입니다.

기존에는 출석을 주차별 엑셀 파일로 따로 관리했기 때문에 여러 주차의 출석 횟수를 합산하거나, 출석 부족자를 확인하는 과정이 번거로웠습니다. 이 프로그램은 Notion에 만들어둔 출석부 데이터를 Python으로 불러와 사람별 총출석 횟수, 부족 횟수, 초과 횟수를 자동으로 계산합니다.

처음에는 출석, 일정, 작업 기록, 지식 공유를 모두 포함하는 협업 보조 시스템으로 기획했지만, 프로젝트 범위가 너무 넓다고 판단하여 현재 버전에서는 **출석 관리 자동화 기능**에 집중했습니다. 초기 기획안은 별도 문서로 옮겨두었고, 현재 README는 최종 구현한 출석 관리 기능을 중심으로 작성했습니다.

프로그램 실행 후에는 전체 출석 결과를 정리한 엑셀 파일, 경고 대상자를 정리한 텍스트 파일, 그리고 출석 현황을 보기 좋게 확인할 수 있는 HTML 파일이 생성됩니다.

---

## 주요 기능

### 1. Notion 출석부 데이터 불러오기

* Notion에 작성된 주차별 출석부 데이터를 불러옵니다.
* 여러 개의 출석부 DB ID를 `.env` 파일에 입력하여 여러 주차를 함께 계산할 수 있습니다.
* `find_db.py`를 이용해 연결된 Notion 데이터베이스 ID를 확인할 수 있습니다.

### 2. 출석 횟수 자동 계산

* 사람별 총출석 횟수를 계산합니다.
* 여러 주차에 같은 이름이 있으면 출석 횟수를 합산합니다.
* 다음과 같은 열을 출석 항목으로 인식합니다.

  * `월`, `화`, `수`, `목`, `금`, `토`, `일`
  * `금A`, `금B`, `토C`처럼 요일 뒤에 A, B, C가 붙은 열
  * `청소`가 포함된 열
  * `노션`이 포함된 열

### 3. 부족/초과 횟수 계산

* 기준 출석 횟수와 비교하여 부족 횟수를 계산합니다.
* 기준 출석 횟수보다 많이 출석한 경우 초과 횟수를 계산합니다.
* 부족 횟수가 2회 이상인 사람은 경고 대상자로 분류합니다.

### 4. 결과 파일 생성

프로그램 실행 후 다음 파일이 생성됩니다.

| 파일명         | 설명                                  |
| ----------- | ----------------------------------- |
| `출석결과.xlsx` | 전체 부원의 총출석, 부족 횟수, 초과 횟수를 정리한 엑셀 파일 |
| `경고공지.txt`  | 경고 대상자를 정리한 공지용 텍스트 파일              |
| `출석현황.html` | 출석 결과를 웹페이지처럼 볼 수 있는 HTML 파일        |

## 실행 방법

### 1. 필요한 라이브러리 설치

아래 명령어로 필요한 라이브러리를 설치합니다.

`pip install requests pandas python-dotenv openpyxl`

---

### 2. `.env` 파일 만들기
docs 폴더에 .env, find_db_py, attendance 파일을 업로드해두었습니다.
실제 실행을 위해서는 `.env` 파일이 필요합니다.
.env파일이 깔려있는 상태에서 다음 작업을 실행합니다.
`.env.example`을 참고하여 다음과 같이 작성합니다.

| 항목                    | 예시                            |
| --------------------- | ----------------------------- |
| `NOTION_TOKEN`        | `your_notion_token_here`      |
| `NOTION_DATABASE_IDS` | `database_id_1,database_id_2` |
| `REQUIRED`            | `12`                          |

각 항목의 의미는 다음과 같습니다.

| 항목                    | 설명                   |
| --------------------- | -------------------- |
| `NOTION_TOKEN`        | Notion API 접근 토큰     |
| `NOTION_DATABASE_IDS` | 계산할 출석부 데이터베이스 ID 목록 |
| `REQUIRED`            | 기준 출석 횟수             |

---

### 3. Notion DB ID 확인

먼저 `find_db.py`를 실행하여 현재 토큰으로 접근 가능한 Notion 데이터베이스 목록을 확인합니다.

`python find_db.py`

실행 결과 예시는 다음과 같습니다.

| 제목         | ID        |
| ---------- | --------- |
| 5월 3주차 출석부 | abc123... |
| 5월 4주차 출석부 | def456... |

출력된 DB ID 중 계산에 사용할 출석부 ID를 복사하여 `.env` 파일의 `NOTION_DATABASE_IDS`에 입력합니다.

---

### 4. 출석 계산 실행

`python attendance.py`

실행 결과 예시는 다음과 같습니다.

| 번호 | 이름  | 총출석 | 부족 | 초과(횟수) |
| -: | --- | --: | -: | -----: |
|  1 | 김규원 |  10 |  2 |      0 |
|  2 | 김기동 |  12 |  0 |      0 |
|  3 | 김성겸 |  16 |  0 |      4 |

실행이 완료되면 다음과 같은 메시지가 출력됩니다.

`저장 완료: 출석결과.xlsx, 경고공지.txt, 출석현황.html`

## 핵심 동작 방식

프로그램은 다음 순서로 동작합니다.

1. `.env` 파일에서 Notion 토큰, DB ID 목록, 기준 출석 횟수를 읽습니다.
2. Notion API를 이용해 출석부 데이터베이스를 하나씩 불러옵니다.
3. 각 행에서 부원 이름을 읽습니다.
4. 출석으로 인정되는 체크박스 열을 확인합니다.
5. 체크된 개수를 세어 개인별 출석 횟수를 계산합니다.
6. 여러 주차에 같은 이름이 있으면 출석 횟수를 합산합니다.
7. 기준 출석 횟수와 비교하여 부족 횟수와 초과 횟수를 계산합니다.
8. 결과를 엑셀, 텍스트, HTML 파일로 저장합니다.

## 개발 중 겪은 문제와 해결

### 1. CSV 파일 방식의 불편함

초기 프로토타입은 Notion에서 CSV 파일을 내보내고, 그 파일을 Python으로 읽는 방식이었습니다. 하지만 매번 CSV 파일을 다운로드하고 압축을 푼 뒤 파일 경로를 코드에 직접 넣어야 해서 번거로웠습니다.

이를 해결하기 위해 Notion API를 사용하여 Python이 출석부 데이터를 직접 읽도록 수정했습니다.

### 2. 출석 열 이름 문제

처음에는 `월`, `화`, `수`, `목`, `금`, `토`, `일`처럼 단순한 요일 열만 읽도록 만들었습니다. 그러나 실제 출석부에는 `금A`, `금B`, `토C`, `수 청소`, `노션` 같은 열도 있었습니다.

그래서 정규표현식을 사용해 `금A`, `토B` 같은 열도 출석으로 인식하도록 했고, 열 이름에 `청소` 또는 `노션`이 포함되어 있으면 추가 출석 항목으로 계산하도록 수정했습니다.

### 3. DB ID 확인 문제

Notion API를 사용하려면 출석부 데이터베이스 ID가 필요합니다. 하지만 주차별 출석부가 여러 개라서 DB ID를 일일이 찾는 과정이 번거로웠습니다.

이를 해결하기 위해 `find_db.py`를 만들었습니다. 이 파일은 현재 Notion 토큰으로 접근 가능한 데이터베이스 제목과 ID를 출력해줍니다.

## 제작 목적

천차많별 동아리는 자동차 제작 대회를 준비하는 동아리로, 여러 명의 임원단과 부원들이 매주 작업에 참여한다. 기존에는 출석 관리를 주차별 엑셀 파일로 따로 작성하고 공유하는 방식으로 진행했다.

하지만 매주 다른 엑셀 파일이 올라오다 보니 이전 주차와 합산해서 확인하기 어렵고, 누가 몇 번 출석했는지 한눈에 보기 힘들었다. 특히 출석 횟수가 부족한 사람이나 기준보다 초과해서 출석한 사람을 매번 직접 계산해야 해서 관리 시간이 많이 들었다.

이러한 문제를 해결하기 위해 Notion에 주차별 출석부를 만들고, Python 프로그램을 이용해 출석 데이터를 자동으로 계산하는 프로그램을 제작하고자 한다. 현재 버전에서는 동아리 운영에 필요한 여러 기능 중 가장 실제로 필요하고 바로 사용할 수 있는 **출석 관리 자동화 기능**에 집중하였다.
