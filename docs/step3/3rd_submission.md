# 3단계 개발 진행 보고

## 1. 구현한 기능

이번 단계에서는 출석 관리 자동화 프로그램의 기본 프로토타입을 구현하였다.

구현한 기능은 다음과 같다.

- Notion API를 이용해 출석부 데이터베이스 읽기
- 여러 주차 출석부 데이터 합산
- 사람별 총출석 횟수 계산
- 기준 출석 횟수 대비 부족 횟수와 초과 횟수 계산
- 부족 횟수가 일정 기준 이상인 사람을 경고 대상자로 분류
- 전체 결과를 `출석결과.xlsx` 파일로 저장
- 경고 대상자를 `경고공지.txt` 파일로 저장
- `find_db.py`를 이용해 연결된 Notion DB ID 확인

---

## 2. 진행 중 겪은 문제

처음에는 Notion 출석부를 CSV 파일로 내보낸 뒤, Python에서 CSV 파일을 읽어 출석 횟수를 계산하는 방식으로 프로토타입을 만들었다. 하지만 이 방식은 매번 CSV 파일을 다운로드하고 압축을 풀어야 했으며, 파일 경로를 코드에 직접 넣어야 해서 사용하기 불편했다.

또한 초기 코드에서는 출석 열을 `월`, `화`, `수`, `목`, `금`, `토`, `일`처럼 단순한 요일 이름만 기준으로 읽도록 만들었다. 그런데 실제 출석부에는 `금A`, `금B`, `토C`처럼 시간대를 나눈 열도 있었고, `수 청소`, `노션`처럼 일반 요일 출석이 아닌 추가 활동 항목도 있었다. 이 때문에 일부 체크박스가 출석 횟수에 포함되지 않는 문제가 발생했다.

Notion API 방식으로 바꾼 뒤에는 각 출석부 데이터베이스의 DB ID를 직접 찾아서 `.env` 파일에 넣어야 하는 문제가 있었다. 주차별 출석부가 여러 개라서 매번 DB ID를 수동으로 찾는 과정이 번거로웠고, 페이지 ID와 데이터베이스 ID를 헷갈리면 데이터를 읽지 못하는 오류도 발생했다.

---

## 3. 해결 방법

CSV 파일을 직접 읽는 방식 대신 Notion API를 사용하여 Python 프로그램이 출석부 데이터를 직접 가져오도록 수정하였다. 이를 통해 CSV 다운로드, 압축 해제, 파일 경로 수정 과정을 줄일 수 있었다.

출석 열을 판단하는 방식도 수정하였다. 단순히 `월`, `화`, `수` 같은 열만 읽는 것이 아니라, 정규표현식을 사용하여 `금A`, `금B`, `토C`처럼 요일 뒤에 A, B, C가 붙은 열도 출석으로 인식하도록 했다. 또한 열 이름에 `청소` 또는 `노션`이 포함되어 있으면 추가 출석 항목으로 계산하도록 조건을 추가했다.

DB ID를 찾는 문제를 해결하기 위해 `find_db.py` 파일을 따로 만들었다. 이 파일은 Notion API 토큰으로 접근 가능한 데이터베이스 목록을 출력해준다. 사용자는 출력된 데이터베이스 제목과 ID를 보고 필요한 출석부 DB ID를 `.env` 파일에 추가할 수 있다.

---

## 4. 현재 동작 방식

현재 프로그램은 다음 순서로 동작한다.

1. `.env` 파일에서 Notion API 토큰, 출석부 DB ID 목록, 기준 출석 횟수를 읽는다.
2. `NOTION_DATABASE_IDS`에 적힌 출석부 DB를 하나씩 불러온다.
3. 각 출석부에서 사람 이름과 체크박스 값을 읽는다.
4. 출석으로 인정되는 열의 체크박스 개수를 센다.
5. 여러 주차에 같은 이름이 있으면 출석 횟수를 합산한다.
6. 기준 출석 횟수와 비교하여 부족 횟수와 초과 횟수를 계산한다.
7. 결과를 터미널에 출력하고, 엑셀 파일과 텍스트 파일로 저장한다.

---

## 5. 남은 작업

현재 프로토타입은 출석 계산과 결과 파일 저장까지 동작한다. 이후에는 결과를 더 보기 좋게 만들기 위해 HTML 결과 페이지를 생성하는 기능을 추가할 예정이다.

추가로 시간이 된다면, 계산 결과를 다시 Notion 페이지에 자동으로 올리는 기능도 검토할 계획이다.
