# 2단계 설계 문서
# 천차 Garage - 출석 관리 자동화 프로그램

## 1. 프로젝트 개요

### 프로젝트 이름
천차 Garage - 출석 관리 자동화 프로그램

### 한 문장 설명
자동차 제작 동아리의 출석부를 자동으로 계산하여 총출석, 부족 횟수, 초과 횟수를 정리해주는 Python 프로그램이다.

---

## 2. 제작 배경

천차많별 동아리는 자동차 제작 대회를 준비하는 동아리로, 매주 여러 명의 부원들이 작업에 참여한다. 기존에는 출석 관리를 주차별 엑셀 파일로 따로 작성하고 공유하는 방식으로 진행했다. 하지만 매주 다른 엑셀 파일이 올라오다 보니 이전 주차와 합산해서 확인하기 어렵고, 누가 몇 번 출석했는지 한눈에 보기 힘들었다.

특히 출석 횟수가 부족한 사람이나 기준보다 초과해서 출석한 사람을 매번 직접 계산해야 해서 관리 시간이 많이 들었다. 또한 이월 횟수나 부족 횟수를 사람이 직접 확인하다 보면 실수가 생길 가능성도 있었다.

이 문제를 해결하기 위해 먼저 Notion에 주차별 출석부를 만들고, Python으로 출석 데이터를 계산하는 방식을 시도하였다. 초기 프로토타입은 Notion에서 CSV 파일을 내보낸 뒤 그 파일을 Python으로 읽어 출석 횟수를 계산하는 방식이었다. 하지만 이 방식도 매번 CSV를 다운로드하고 압축을 풀고 파일 경로를 수정해야 해서 번거로웠다.

따라서 최종적으로는 Notion 출석부 데이터를 Python 프로그램이 직접 읽고, 여러 주차의 출석 횟수를 자동 합산하는 방식으로 개선하고자 한다.

## 4. 데이터 구조 설계

### 4.1 Notion 출석부 데이터

Notion 출석부는 한 행이 한 명의 부원을 의미한다.

예시:

| 이름 | 월 | 화 | 수 | 금A | 금B | 수 청소 | 노션 |
|---|---|---|---|---|---|---|---|
| 김규원 | 체크 |  | 체크 |  | 체크 | 체크 |  |
| 홍서연 |  | 체크 | 체크 | 체크 |  |  | 체크 |

각 열은 다음과 같이 처리한다.

| 열 이름 | 처리 방식 |
|---|---|
| 이름 | 부원 이름으로 사용 |
| 월, 화, 수, 목, 금, 토, 일 | 출석 항목으로 계산 |
| 금A, 금B, 토A, 일C 등 | 출석 항목으로 계산 |
| 청소가 포함된 열 | 출석 항목으로 계산 |
| 노션이 포함된 열 | 출석 항목으로 계산 |
| 그 외 열 | 계산에서 제외 |

### 4.2 설정 파일 데이터

프로그램 실행에 필요한 값은 `.env` 파일에 저장한다.

```env
NOTION_TOKEN=노션_API_토큰
NOTION_DATABASE_IDS=출석부DB_ID1,출석부DB_ID2
REQUIRED=12
```

| 항목 | 의미 |
|---|---|
| NOTION_TOKEN | Notion 데이터를 읽기 위한 인증 토큰 |
| NOTION_DATABASE_IDS | 계산할 출석부 데이터베이스 ID 목록 |
| REQUIRED | 기준 출석 횟수 |

### 4.3 Python 내부 데이터

처음에는 각 주차별 출석 결과를 리스트에 저장한다.

```python
rows = [
    ["김규원", 4],
    ["홍서연", 5],
    ["김규원", 6]
]
```

이후 이름이 같은 사람끼리 합산한다.

예시:

| 이름 | 총출석 |
|---|---:|
| 김규원 | 10 |
| 홍서연 | 5 |

최종 결과표는 다음과 같은 형태로 만든다.

| 번호 | 이름 | 총출석 | 부족 | 초과(횟수) |
|---:|---|---:|---:|---:|
| 1 | 김규원 | 10 | 2 | 0 |
| 2 | 김기동 | 12 | 0 | 0 |
| 3 | 김성겸 | 16 | 0 | 4 |

## 5. 함수별 의사코드

현재 코드는 설명하기 쉽도록 큰 기능을 순서대로 실행하는 구조로 설계한다. 이후 필요하면 함수로 분리할 수 있다.

현재 프로그램은 크게 두 개의 파일로 구성한다.
- `find_db.py`: Notion 연결이 된 데이터베이스 목록을 확인하고 DB ID를 찾는 보조 프로그램
- `attendence.py`: 실제 출석 횟수를 계산하고 결과 파일을 만드는 메인 프로그램

### 5.1 find_db.py 의사코드

```text
.env 파일에서 Notion API 토큰을 읽는다
Notion API에 접근 가능한 데이터베이스 목록을 요청한다

요청이 성공하면 데이터베이스 제목과 ID를 출력한다

요청이 실패하면 오류 내용을 출력한다

사용자는 출력된 DB ID 중 필요한 출석부 ID를 복사해 .env 파일에 입력한다
```
### 5.2 attendence.py - 설정값 읽기

```text
.env 파일을 읽는다
노션 토큰을 가져온다
출석부 DB ID 목록을 가져온다
기준 출석 횟수를 가져온다
DB ID 목록을 쉼표 기준으로 나눈다
```

### 5.3 attendence.py - 출석부 데이터 읽기

```text
전체 출석 정보를 저장할 빈 리스트를 만든다

출석부 DB ID를 하나씩 반복한다
    Notion API로 해당 출석부 데이터를 요청한다

    요청이 실패하면
        오류 내용을 출력하고 다음 DB로 넘어간다

    요청이 성공하면
        출석부의 각 행을 하나씩 읽는다
```

### 5.4 attendence.py - 출석 횟수 계산

```text
각 사람에 대해
    이름을 읽는다
    출석 횟수를 0으로 시작한다

    모든 열을 확인한다
        열 이름이 월, 화, 수, 금A, 토B 같은 형식이면 출석 열로 본다
        열 이름에 청소 또는 노션이 포함되어도 출석 열로 본다

        해당 열이 체크되어 있으면
            출석 횟수를 1 증가시킨다

    이름과 출석 횟수를 리스트에 저장한다
```

### 5.5 attendence.py - 합산 및 부족/초과 계산

```text
저장된 리스트를 표 형태로 바꾼다
이름이 같은 사람끼리 출석 횟수를 합산한다

각 사람에 대해
    부족 = 기준 출석 횟수 - 총출석
    부족이 음수이면 0으로 처리한다

    초과 = 총출석 - 기준 출석 횟수
    초과가 음수이면 0으로 처리한다

이름순으로 정렬한다
번호를 붙인다
```

### 5.6 attendence.py - 결과 저장

```text
전체 결과를 출석결과.xlsx로 저장한다

부족 횟수가 2 이상인 사람을 경고 대상자로 분류한다
경고공지.txt 파일을 만든다

경고 대상자가 있으면
    이름과 부족 횟수를 저장한다

경고 대상자가 없으면
    경고 대상자가 없다고 저장한다
```

## 6. 화면 흐름 설계

### 6.1 실행 전 준비

사용자는 먼저 Notion 출석부 데이터베이스에 `천차출석부자동화` 연결을 추가한다.  
그리고 `.env` 파일에 Notion API 토큰과 기준 출석 횟수를 입력한다.

```env
NOTION_TOKEN=노션_API_토큰
REQUIRED=기준_출석_횟수
```

### 6.2 출석부 DB ID 찾기

출석부 데이터베이스 ID를 확인하기 위해 `find_db.py`를 먼저 실행한다.

```bash
python find_db.py
```

실행하면 현재 연결된 Notion 데이터베이스 목록이 출력된다.

```text
[접근 가능한 데이터베이스 목록]

제목: 5월 3주차 출석부
ID: abc123...

제목: 5월 4주차 출석부
ID: def456...
```

사용자는 여기서 계산에 사용할 출석부 DB ID를 복사해 `.env` 파일에 추가한다.

```env
NOTION_TOKEN=노션_API_토큰
NOTION_DATABASE_IDS=5월3주차_DB_ID,5월4주차_DB_ID
REQUIRED=기준_출석_횟수
```

### 6.3 출석 계산 프로그램 실행

출석부 DB ID와 기준 출석 횟수를 설정한 뒤, 실제 출석 계산 프로그램인 `attendence.py`를 실행한다.

```bash
python attendence.py
```

```text

  1 김규원   10   2       0
  2 김기동   12   0       0
  3 김성겸   16   0       4

저장 완료: 출석결과.xlsx, 경고공지.txt
```

| 출석현황.html | 추후 추가할 웹페이지 형태 결과 파일 |
